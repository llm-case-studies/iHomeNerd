# iHomeNerd Dashboard UI Spec

**Date:** 2026-04-13
**Status:** Build spec for AI Studio / Codex
**Stack:** Single-page app served by the existing FastAPI backend. Plain HTML + CSS + vanilla JS (or htmx + Alpine.js). No build step, no npm, no bundler. Files live in `backend/app/static/` and `backend/app/templates/`.

## Design principles

1. **One HTML page, multiple panels** — tab-style navigation, no routing framework
2. **Dark theme by default** — matches terminal aesthetic, easy on eyes for home use
3. **Mobile-friendly** — works on phone browsers over LAN (responsive grid)
4. **Zero build step** — edit HTML/CSS/JS, refresh browser. No webpack, no Vite, no node_modules
5. **Backend is already built** — all features call existing REST endpoints at `/v1/*`, `/health`, `/capabilities`

## Layout

```
┌─────────────────────────────────────────────────┐
│  🏠 iHomeNerd          [status pill]   [⚙ gear] │
├──────┬──────┬──────┬──────┬──────┬──────────────┤
│ Chat │ Talk │ Docs │ Translate │ System │        │
├──────┴──────┴──────┴──────┴──────┴──────────────┤
│                                                   │
```

- **Header**: product name, status pill (green = Ollama + models OK, amber = degraded, red = offline), gear icon for settings
- **Tab bar**: Chat, Talk, Docs, Translate, System
- **Panel area**: full remaining height, scrollable

## Panels

### 1. Chat

General-purpose local AI chat.

**Layout:**
- Message list (scrollable, bottom-anchored)
- Input area at bottom: textarea + send button
- Messages show user (right-aligned) and assistant (left-aligned)
- Assistant messages render markdown (bold, lists, code blocks)

**API:** `POST /v1/chat` with `{ "messages": [{ "role": "user", "content": "..." }] }`
- Maintain conversation history in JS (array of messages)
- Send full history each turn
- "New chat" button clears history

### 2. Talk (Voice)

Voice conversation — the ASR → brain → TTS pipeline in a UI.

**Layout:**
- Large center microphone button (press-and-hold or toggle)
- Conversation transcript below (alternating user / agent bubbles)
- Each agent bubble has a small speaker icon to replay audio
- Voice selector dropdown (Kokoro voices, grouped by language)
- Language selector for target language

**Flow:**
1. User presses mic → browser `MediaRecorder` captures audio (WebM/opus)
2. On release → `POST /v1/transcribe-audio` (multipart, file + language)
3. Display user text in transcript
4. Send transcribed text to `POST /v1/chat` (or `/v1/dialogue-turn` if in scenario mode)
5. Take agent response text → `POST /v1/synthesize-speech` with `{ "text": "...", "targetLang": "..." }`
6. Play returned WAV audio via `<audio>` element or AudioContext
7. Display agent text + gloss in transcript

**Scenario mode toggle:**
- Off = general chat (uses `/v1/chat`)
- On = bounded dialogue (uses `/v1/dialogue-session` + `/v1/dialogue-turn`)
- When on, show scenario setup: prompt textarea, role names, language, difficulty, turn budget
- Show turn counter and "end scenario" button

### 3. Docs (Document Copilot)

Upload documents, ask questions, browse extracted knowledge.

**Note:** Backend endpoints for this (`/v1/docs/*`) are NOT yet built. This panel should show a "coming soon" state initially, but the UI should be ready for when the backend catches up.

**Layout (ready state):**
- Left sidebar: document/collection list
- Right area: chat-style Q&A against selected documents
- Upload button (drag-and-drop zone) at top of sidebar
- Each document shows: name, page count, date added, status (indexed / pending)

**Layout (coming soon state):**
- Centered card: "Document Copilot — coming soon"
- Brief description: "Upload tax forms, bills, manuals. Ask questions. Get answers from your own documents."
- List planned capabilities: tax copilot, bill explainer, manual Q&A

### 4. Translate

Simple translation tool.

**Layout:**
- Two-column or stacked (mobile): source panel and target panel
- Source: textarea + language dropdown (auto-detect default)
- Target: read-only textarea + language dropdown
- Translate button between them (or auto-translate on pause)
- Swap languages button
- Copy result button

**API:** `POST /v1/translate` with `{ "text": "...", "source": "auto", "target": "es" }`

**Language list:** EN, ZH, JA, KO, ES, FR, DE, IT, PT, RU, TR, UK, HI, AR (start with what Gemma handles well)

### 5. System

Status dashboard showing what the Nerd is doing.

**Layout — top cards row:**
- Ollama status (green/red + model count)
- TTS status (Kokoro loaded + voice count)
- ASR status (Whisper model + size)
- Uptime

**Layout — capabilities table:**
- Rows from `/capabilities` endpoint
- Columns: capability name, available (checkmark/cross), model, tier
- Green rows for available, gray for unavailable

**Layout — models section:**
- List from `/health` → `models` map
- Show model name, tier, and which capabilities use it

**Layout — active sessions:**
- List from `/sessions`
- Show session ID (truncated), app, purpose, turn count, age

**Layout — connected apps section:**
- Show any PronunCo or other apps that have recently probed `/health`
- (Future: show paired LAN clients)

## Styles

### Color palette (dark theme)

```css
--bg-primary: #0f1117;       /* page background */
--bg-surface: #1a1d27;       /* cards, panels */
--bg-input: #252830;         /* input fields */
--border: #2e3140;           /* subtle borders */
--text-primary: #e4e6eb;     /* main text */
--text-secondary: #8b8fa3;   /* labels, hints */
--accent: #4f8cff;           /* buttons, links, active tab */
--accent-hover: #6ba0ff;
--success: #34d399;          /* green status */
--warning: #fbbf24;          /* amber status */
--error: #f87171;            /* red status */
--user-bubble: #1e3a5f;      /* user message bg */
--agent-bubble: #1a2332;     /* agent message bg */
```

### Typography

- System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- Monospace for code/system info: `"JetBrains Mono", "Fira Code", "Consolas", monospace`
- Base size: 15px
- Chat messages: 14px
- Tab labels: 13px uppercase, letter-spacing 0.5px

### Component patterns

- **Cards**: `bg-surface`, 8px radius, 1px border, 16px padding
- **Buttons**: `accent` bg, white text, 6px radius, 10px 16px padding
- **Inputs/textareas**: `bg-input`, 1px border, 6px radius, focus ring with accent
- **Status pills**: small rounded badge with colored dot + text
- **Tabs**: underline-style active indicator, no background change

## API endpoints summary

Already built and available:

| Endpoint | Method | Panel | Purpose |
|---|---|---|---|
| `/health` | GET | System, Header | Status check |
| `/capabilities` | GET | System | Capability registry |
| `/sessions` | GET | System | Active sessions |
| `/v1/chat` | POST | Chat, Talk | General chat |
| `/v1/translate` | POST | Translate | Translation |
| `/v1/summarize` | POST | Docs (future) | Text summarization |
| `/v1/lesson-extract` | POST | (PronunCo) | Lesson extraction |
| `/v1/dialogue-session` | POST | Talk (scenario) | Start scenario |
| `/v1/dialogue-turn` | POST | Talk (scenario) | Advance turn |
| `/v1/synthesize-speech` | POST | Talk | TTS → WAV |
| `/v1/transcribe-audio` | POST | Talk | ASR → text |
| `/v1/voices` | GET | Talk | List TTS voices |

2. Chat panel (most universally useful, simplest to build)
3. Translate panel (quick win, already works)

**Phase 2 — Voice:**
4. Talk panel (mic + audio playback + transcript)

**Phase 3 — Documents:**
5. Docs panel (needs backend work first)

## Build instructions for AI Studio

1. Create `backend/app/templates/index.html` with all panel markup
2. Create `backend/app/static/style.css` with the dark theme
3. Create `backend/app/static/app.js` with panel switching and API calls
4. The FastAPI server already serves `/static/*` and renders templates via Jinja2
5. Add a `GET /` route in `main.py` that renders `index.html`
6. Test by opening `http://localhost:17777/` in a browser

No npm, no build step, no framework. Just HTML + CSS + JS files served by the existing FastAPI backend.
