## 5. Gotchas

These cost real time the first time around. Document well.

### Two macOS users on the Mac mini

- `alex` — interactive desktop login. **Apple ID + Personal Team + dev cert live here.** All ssh-driven device builds must run as alex.
- `funhome` — early ssh experiments landed here. Don't bother. Default in old Makefile was funhome; current default is alex.
- The same key (`~/.ssh/funhome-local.pub` on the Linux box) is in `authorized_keys` for both users.

### Keychain unlock is per-session

- macOS keychain unlock state is per process tree. Each new ssh session sees the keychain re-locked.
- Fix: `make remote-device-build` unlocks the login keychain inline using `$IHN_KEYCHAIN_PW` env var before running `xcodebuild`. User must export the password once per shell session.
- A one-time partition-list grant was done so codesign can access the cert without an interactive prompt. Recipe (already applied):
  ```zsh
  printf 'Mac password: '; read -rs PW; echo
  security unlock-keychain -p "$PW" ~/Library/Keychains/login.keychain-db
  security set-key-partition-list -S apple-tool:,apple: -s -k "$PW" ~/Library/Keychains/login.keychain-db
  security set-keychain-settings ~/Library/Keychains/login.keychain-db
  unset PW
  ```
  This grants `apple-tool` (codesign, xcodebuild) permission to access the cert non-interactively, and disables auto-lock timeout. Per-session unlock is still needed though — see above.

### Xcode 26.3 destination matcher requires booted simulator

- xcodebuild for `-destination 'platform=iOS Simulator,name=...'` errors with "no destination matching" if the named simulator isn't already booted, even if simctl lists it.
- Fix: Makefile's `remote-build` target now boots the simulator first via `xcrun simctl boot "$(SIM_DEVICE)"` (idempotent — `|| true`).

### Brew install flips xcode-select

- `/bin/bash -c "$(curl ...install.sh)"` for Homebrew runs `sudo xcode-select --switch /Library/Developer/CommandLineTools` as a side effect, breaking xcodebuild ("active developer directory is a command line tools instance").
- Fix: Makefile sets `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer` for every ssh-driven xcodebuild call. xcode-select state irrelevant to our builds.

### Mac mini Wi-Fi sleep / IP rotation

- Mac mini drops off LAN aggressively when display sleeps. Set System Settings → Energy → "Prevent automatic sleeping" + "Wake for network access".
- IP rotates via DHCP (last seen `192.168.0.221`, was `192.168.0.42` earlier). Use mDNS hostname `mac-mini-m1.local` everywhere, not raw IPs.

### iOS 26.2 SDK / iOS 26.3 simulator runtime mismatch

- Xcode 26.3 ships with iOS 26.2 SDK bundled, but the installed simulator runtime is iOS 26.3. `xcrun simctl list devices available` shows iPhone 17-class only (no 15, no 12). Doesn't affect builds — apps built against iOS 26.2 SDK run fine on 26.3 sim.
- For real-device testing of older iPhones, use the actual phones (no need for older simulators).

## 8. Resume-from-cold checklist

If you're starting fresh on the Mac mini:

1. Repo: clone or sync the Linux repo. The simplest path is `git pull` if the Linux side committed, or rsync `./` from Linux.
2. Verify Xcode is the active dev dir: `xcode-select -p` should print `/Applications/Xcode.app/Contents/Developer`. If not, `sudo xcode-select -s /Applications/Xcode.app/Contents/Developer`.
3. Check codesigning identity: `security find-identity -v -p codesigning` should list `Apple Development: alexsudakov@prodigy.net`.
4. If on Mac-native shell, you don't need `IHN_KEYCHAIN_PW` or the per-session unlock — keychain is unlocked by your interactive login.
5. Open `~/ihn-home-ios/IhnHome.xcodeproj` in Xcode. Pick the iPhone in the destination dropdown. Cmd-R to build + run.
6. To keep the Linux-side `make` flow working in parallel, just don't delete `~funhome/ihn-home-ios` — both can coexist.
